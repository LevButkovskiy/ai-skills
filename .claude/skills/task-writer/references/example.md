# Эталонный пример задачи

## Input
Разработчик описал проблему в чате:
- Сервис MaxiBots сутки перезапускался в loop
- Rabbit был жив, но соединение не восстанавливалось
- Ручной рестарт помог
- Причина: amqplib не умеет реконнектиться, ошибки не обрабатывались

## Output

**Стабилизация подключения к RabbitMQ в сервисе MaxiBots**

**Причина**
Сервис уходил в бесконечный цикл перезапусков при разрыве соединения с RabbitMQ. При автоматическом рестарте процесс мгновенно падал снова — соединение не восстанавливалось. Только ручной рестарт (с паузой) решал проблему. Корень проблемы — `amqplib` не умеет переподключаться сам, а необработанные ошибки на канале крашили весь процесс.

**Задача**
Заменить прямое использование `amqplib` на `amqp-connection-manager` в файле `core/amqp.js`. Переписать `createChannel` и `registerWebhooksReciver` под новую библиотеку.

**Что использовать**
Библиотека `amqp-connection-manager` (npm).

**Ожидаемый результат**
- При разрыве соединения с RabbitMQ сервис автоматически переподключается без краша процесса
- Каналы и очереди пересоздаются после реконнекта автоматически
- Изменения только в `core/amqp.js`, остальной код (subscribers, moduleMaxiBots) не трогаем
