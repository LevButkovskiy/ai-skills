# Create Prompt

Skill для создания структурированных промтов в XML формате для Claude Code агентов.

## Процесс

### Шаг 1 — Понять задачу

Извлеки из контекста разговора:
- Что должен сделать агент
- Какие файлы или данные у него будут на входе
- Какой результат ожидается на выходе
- Одноразовый промт или переиспользуемый

Если чего-то не хватает — задай один уточняющий вопрос, самый важный. Не задавай несколько вопросов сразу.

### Шаг 2 — Уточнить неясности

Перед написанием промта убедись что знаешь:
- Входные данные: какие файлы читает агент, что уже готово к моменту запуска
- Выходные данные: файл, папка, конкретный формат
- Зависимости: на какие другие файлы или результаты опирается задача
- Есть ли соглашения проекта (CLAUDE.md, conventions.md), которые агент должен учитывать

### Шаг 3 — Написать промт

Структура XML промта:

```xml
<context>
Что агент должен знать о проекте и ситуации.
Ссылки на файлы которые нужно прочитать перед стартом.
</context>

<task>
Одно чёткое предложение — что нужно сделать.
</task>

<instructions>
Пронумерованные шаги. Первый шаг всегда — прочитать нужные файлы.
Конкретные действия, не абстрактные цели.
</instructions>

<rules>
Ограничения и запреты если есть.
</rules>

<format>
Куда сохранить результат и в каком виде.
</format>

<done_criteria>
- [ ] Проверяемые пункты завершённости
</done_criteria>
```

## Правила написания

- `<rules>` и `<done_criteria>` — опциональны, добавляй только если нужны
- Первый шаг в `<instructions>` всегда — прочитать входные файлы
- Пиши императивно: "Прочитай", "Найди", "Создай"
- Не объясняй агенту зачем — только что делать
- `<done_criteria>` должны быть проверяемыми, не субъективными

## После написания

Покажи промт пользователю и спроси:
- Всё ли точно отражает задачу
- Не хватает ли какого-то контекста

Итерируй до подтверждения.
